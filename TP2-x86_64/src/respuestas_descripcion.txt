3. 
La fase 3 del binary bomb espera como entrada dos valores:

Una palabra que debe estar presente en el archivo palabras.txt.

Un número entero que corresponde al resultado de ejecutar una función de búsqueda binaria recursiva sobre esa palabra. La función que se encarga de esto se llama cuenta, y se encuentra desensamblada a partir de la dirección 0x401f98.La función cuenta implementa una búsqueda binaria recursiva, pero con una diferencia importante: Cada vez que evalúa un índice intermedio (mid), suma ese valor al resultado final.

Entonces, para simular estas operaciones, se creó un archivo de python que simulara la función 'cuenta'. De allí se obtuvo un número posible
para cada palabra de 'palabras.txt'. Estas opciones obtenidas por aquella función se encuentran en el archivo 'phase_3_inputs.txt'.

La instrucción clave que refleja el comportamiento 'especial' de esta búsqueda binaria es '401fe1: add    ebx, eax'. Acá:
- ebx contiene el valor de mid de la llamada actual,
- eax contiene el resultado acumulado de la recursión,
Estos se suman antes de hacer ret.

Si la palabra no se encuentra, se llama a explode_bomb.
Si el número ingresado no coincide con el valor devuelto por cuenta, también explota.


En conclusión, para que la bomba no explote se deben cumplir 3 condiciones:
- La palabra debe estar en la lista palabras.txt.
- El número ingresado debe ser exactamente igual al resultado de cuenta, es decir, la suma de los valores de mid en cada llamada recursiva.
- Este número debe ser menor o igual a 9999 (hex 0x270f), ya que se verifica con cmp eax, 0x270f.
